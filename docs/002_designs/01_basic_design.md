# 基本設計書

## 1. 目的

毎日の「良かった出来事」を記録することで幸福度を高める。記録体験を最短化し、簡単に見返せるようにする。さらに、全国の記録を掲示板形式で共有できるようにする。

## 2. 対象ユーザー

- 日々の小さな良いことを継続的に記録したいユーザー
- 他人の「良いこと」を見てポジティブな気分になりたいユーザー

## 3. 主要要件（機能要件）

- 1日に記録できる「良かった出来事」は最大3件
- 記録までのステップを最短化（入力→保存が最小手順）
- 記録の振り返りが簡単にできる（カレンダー/日付別一覧など）
- 2chのような掲示板で、全国の良いこと記録を誰でも閲覧可能
- 認証必須（ログインユーザーのみ記録可能）

## 4. 非機能要件（優先度高）

- 入力の高速性（低レイテンシ、軽いUI）
- 直感的な操作性（迷わない導線）
- 掲示板は完全匿名で表示

## 5. 画面一覧（案）

- ホーム（今日の記録 + 入力フォーム）
- 振り返り（カレンダー/日付別一覧）
- 掲示板（全国の良いこと一覧）
- 設定/プロフィール（任意、表示名や公開範囲）

## 6. 機能概要

### 6.1 記録機能

- 当日の記録を最大3件まで登録
- 1件あたり短文入力（140文字上限）
- 入力は1画面で完結

### 6.2 振り返り機能

- 日付単位で自分の記録を一覧表示
- 直近7日/30日などのスナップショット表示

### 6.3 いいこと共有（掲示板）

- すべての公開記録を時系列で表示
- 2ch風のテキスト主体UI（完全匿名）
- フィルタ（今日/今週）や簡易検索は将来拡張

## 7. システム構成（概略）

- フロント: Next.js App Router
- サーバー: Next.js Server Actions / API Routes
- DB: Prisma + Supabase(Postgres)

## 8. データ概要（概略）

- ユーザー（認証必須）
- 日次記録（日付単位、最大3件)
- 記録アイテム（良いことの本文）
- 掲示板表示フラグ（デフォルト公開）
