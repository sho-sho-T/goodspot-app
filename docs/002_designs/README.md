# 002_designs

このディレクトリは「良いこと記録アプリ（Goodspot）」の設計ドキュメント一式を格納します。

## 収録ドキュメント

- `01_basic_design.md`: 基本設計書（目的・要件・画面/機能・全体構成）
- `02_detail_design.md`: 詳細設計書（画面/状態/バリデーション/フロー/権限制御）
- `03_api_spec.md`: API/サーバーアクション仕様（入出力・エラー・認可）
- `04_data_model.md`: データモデル設計（テーブル/リレーション/制約）

## 前提

- 要件はユーザー提供の内容をベースに作成。
- 未決定点は各ドキュメントの「未決定/要確認」に整理。
